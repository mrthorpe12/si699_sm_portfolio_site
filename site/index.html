Extending dynamic web apps

Introduction

Michigan Radio has built a database of speech-to-text transcripts for city council meetings. However, they do not have a way for users to select specific keywords and receive notifications. To rectify this, we built a web application that allows users to search for keywords and receive notifications when these keywords are identified again. We also allowed for users to view their alerts, along with editing their alerts, when they are alerted, and deleting them altogether.

Key Features

Email Alerts

Users will receive email alerts whenever a story containing one or more user-specified keywords is uploaded. Each alert will contain an embedded video or a video link, as well as a summary of the story contents.

Alert Dashboard

Users can access a personalized dashboard where they can create, update, and delete their personalized alerts.

Additional search features

Users can select multiple state and locality values. Users can also specify as many search terms as they would like. Alerts can be created independently of running searches.

Technology Stack

Our technology stack consists of React, Django, PostgreSQL, and Node.js, which allowed us to build a robust, scalable, and responsive web application.

React

  • Used for building the frontend
  • Enabled component-based UI development
  • Improved app performance with virtual DOM

Django

  • Powered the backend with Django REST framework
  • Provided built-in admin interface
  • Ensured security and scalability

PostgreSQL

  • Chosen for its robustness and performance
  • Stored and managed all application data
  • Supported advanced querying capabilities

Node.js

  • Used for server-side JavaScript execution
  • Enabled seamless integration with frontend
  • Supported efficient package management with npm

User Research and Interviews

We conducted user research by organizing interviews with potential users. These interviews provided valuable insights into user needs, preferences, and pain points. This user-centric approach allowed us to identify key features and functionalities that would enhance the user experience and address the challenges faced by our target audience.

High-Fidelity Wireframe Prototypes

Our team designed high-fidelity wireframe prototypes for our application. These prototypes enabled us to visualize the user interface and user experience, ensuring that our app met the needs of our target audience. By iterating on these prototypes, we were able to refine our design and establish a solid foundation for the final implementation.

Constructing the RESTful API with Django

We built a robust and scalable RESTful API using Django to power the back-end of our application. This API enabled seamless communication between the front-end and our data sources, ensuring efficient data retrieval and storage. By using Django, we ensured that our API was reliable, secure, and easily maintainable.

Final Front-End Implementation with React

Our team implemented the front-end of our application using React, a popular and powerful JavaScript framework. This allowed us to create a responsive and dynamic user interface that provided an engaging and seamless user experience. By leveraging React's component-based architecture, we were able to build a maintainable and scalable application that met the needs of our users.
