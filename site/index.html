Introduction

Michigan Radio has built a database of speech-to-text transcripts for city council meetings. However, they do not have a way for users to select specific keywords and receive notifications. To rectify this, we built a web application that allows users to search for keywords and receive notifications when these keywords are identified again. We also allowed for users to view their alerts, along with editing their alerts, when they are alerted, and deleting them altogether.

Technology Stack

Our technology stack consists of React, Django, PostgreSQL, and Node.js, which allowed us to build a robust, scalable, and responsive web application.

React

  • Used for building the frontend
  • Enabled component-based UI development
  • Improved app performance with virtual DOM

Django

  • Powered the backend with Django REST framework
  • Provided built-in admin interface
  • Ensured security and scalability

PostgreSQL

  • Chosen for its robustness and performance
  • Stored and managed all application data
  • Supported advanced querying capabilities

Node.js

  • Used for server-side JavaScript execution
  • Enabled seamless integration with frontend
  • Supported efficient package management with npm
